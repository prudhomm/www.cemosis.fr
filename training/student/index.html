OneFeel


Contexte

Le projet OpenSource Feel++ (http://www.feelpp.org) est une plateforme permettant de faire de la simulation numérique en résolvant en particulier ce qu'on appelle des équations aux dérivées partielles. C'est un framework à la fois pour l'enseignement, la recherche et la R&D en milieu industriel. C'est essentiellement une librairie C++ dont les composants sont assemblés pour faire des simulations. Feel++ s'insère dans un environnement classique de simulation : PreProcessing (e.g. CAO) -> Simulation (Feel++) -> PostProcessing (e.g. Visualisation 2D/3D, films, …). Les applications sont dans le domaine de l'ingénierie classique (mécanique des fluides, structure, thermique,...), du bio-médical (e.g. écoulements sanguins, détection et ablation de tumeur,...), de la physique (simulation de champ magnétique intense).

Des vidéos de simulations sont disponibles sur le site web de Feel++


Projet

Feel++ est une librairie C++ ne comportant aucun composant graphique. L'objectif du projet est de créer une interface pour Feel++ via une approche Client/Serveur (OneLAB) avec une interface graphique qui est Gmsh (http://geuz.org/gmsh/). Gmsh est déjà utilisé par Feel++ pour la CAO et la génération de maillage et dans une moindre mesure pour la visualisation. Il s'agit de mettre en place

  • une interface scriptée entre les applications de Feel++ et l'interface graphique Gmsh à travers OneLAB,
  • une interface native C++ (ou Python, ou les deux) pour les application de Feel++.

OneLAB opère sur modèle client/serveur disposant d'une base de données coté serveur et d'une interface graphique (optionnelle) Gmsh controlant des clients locaux ou distants communicant avec le serveur soit directement en mémoire ou via des sockets TPC/IP.
L'objectif est de pouvoir produire des applications Feel++ tournant soit localement soit sur des calculateurs parallèles distants avec interface graphique pour piloter ces applications et visualiser les résultats.
Enfin nous développons des méthodes numériques pour faire des simulations temps réel paramétrisées par des paramètres physiques et/ou géométriques. Leur particularité est la quasi instantanéité des calculs ce qui ne sera pas le cas pour la plupart des applications standards Feel++. Il s'agira donc aussi de prendre en compte ce type de simulations.
